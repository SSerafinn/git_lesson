Committing changes

Goals

Learn how to commit staged changes with clear and meaningful commit messages.

01

Making a commit

Once you’ve staged your changes, you must create a commit. Every commit requires a commit message—Git will not allow you to commit without one.

Run
git commit -m "Add header to Hello World page"

The -m flag is used to add the commit message directly in the command line.

02

Commit message conventions

Commit messages should be:

  • Clear and descriptive – explain what the commit does;
  • Written in imperative mood – e.g., “Add feature” not “Added feature”;
  • Concise but specific – avoid vague messages like “fix” or “update.”
Good examples 
git commit -m "Add H1 header to Hello World page"
git commit -m "Fix typo in introduction paragraph"
git commit -m "Refactor CSS for navigation bar"
Bad examples 
git commit -m "update"
git commit -m "stuff"
git commit -m "final changes"

Following a proper naming convention makes your project history easier to read and maintain.
