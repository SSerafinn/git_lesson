Staging the changes

1

Goals

Learn to stage changes for the upcoming commits.

2

Adding changes

Now command Git to stage changes. Check the status:

Run
git add hello.html
git status

You will see:

Result 
$ git add hello.html
$ git status
On branch main
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   hello.html

Changes to hello.html have been staged. Git knows about the change, but it isn’t permanent in the repository until committed. The next commit will include these staged changes.

If you decide not to commit the change, git status reminds you that you can run git restore --staged to unstage them.
