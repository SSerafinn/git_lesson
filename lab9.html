LAB 9

History

1

Goals

Learn to view the project’s history.

Getting a list of changes is done with the git log command.

Run
git log

You will see:

Result 
$ git log
commit b7614c1ae... 
Author: Your Name <you@example.com>
Date:   Tue Nov 28 05:51:38 2023 -0600

    Added HTML header

commit 46afaff223...
Author: Your Name <you@example.com>
Date:   Tue Nov 28 05:51:38 2023 -0600

    Added standard HTML page tags

commit 78433de967...
Author: Your Name <you@example.com>
Date:   Tue Nov 28 05:51:38 2023 -0600

    Added h1 tag

commit 5836970eaf...
Author: Your Name <you@example.com>
Date:   Tue Nov 28 05:51:38 2023 -0600

    Initial commit

That’s the list of commits we’ve created so far.

01

One line history

You fully control what git log shows. A handy compact view is single-line:

Run
git log --oneline

You will see:

Result 
$ git log --oneline
b7614c1 Added HTML header
46afaff Added standard HTML page tags
78433de Added h1 tag
5836970 Initial commit
02

Controlling the display of entries

Some useful options for viewing history:

git log --oneline --max-count=2
git log --oneline --since="5 minutes ago"
git log --oneline --until="5 minutes ago"
git log --oneline --author="Your Name"
git log --oneline --all

There are many more. See git log’s manual page for a full list.

03

Getting fancy

A handy way to review changes from the last week (and optionally filter by author) is: 

git log --all --pretty=format:"%h %ad | %s (%an)" --since="7 days ago"
04

The ultimate format of the log

A readable, information-dense format I like is:

Run
git log --pretty=format:"%h %ad | %s %d [%an]" --date=short

It looks like this:

Result 
$ git log --pretty=format:"%h %ad | %s %d [%an]" --date=short
b7614c1 2023-11-28 | Added HTML header (HEAD -> main) [Your Name]
46afaff 2023-11-28 | Added standard HTML page tags [Your Name]
78433de 2023-11-28 | Added h1 tag [Your Name]
5836970 2023-11-28 | Initial commit [Your Name]

Breakdown:

  • --pretty="..." — defines the output format.
  • %h — abbreviated commit hash.
  • %ad — commit date.
  • | — visual separator.
  • %s — commit message.
  • %d — decorations (e.g., branch names or tags).
  • %an — author name.
  • --date=short — short, tidy dates.

To avoid retyping, you can set your default log output:

Run 
git config --global format.pretty "%h %ad | %s %d [%an]"
git config --global log.date short
05

Other tools

GUI tools like gitx (macOS) and gitk (cross-platform) can help explore history visually.
