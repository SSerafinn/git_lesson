Staging and committing

A separate indexing step in Git allows you to split large changes into small commits. For example, say you washed your car and also refilled the windshield washer fluid. These are essentially independent, so it’s better to commit them separately. Otherwise, the log entry may read “Washed the car” while it actually also included the unrelated windshield washer fluid refill.

Suppose you’ve edited three files (a.html, b.html, c.html). You want a.html and b.html in one commit, and c.html in another, since it’s not related.

In theory, you can do the following:

git add a.html
git add b.html
git commit -m "Changes for a and b"
git add c.html
git commit -m "Unrelated change to c"

By separating staging and committing, you can easily customize what goes into a commit.
